143 """ 

144 Attributes defined for Open edX Discussion Topic Context object. 

145 

146 Context for linking the external id for a discussion topic to its associated usage key. 

147

149 title (str): title of the discussion. This field is cached to improve the performance, since otherwise we'd 

150 need to look it up in the course structure each time.

301@attr.s(frozen=True) 

302class ExamAttemptData: 

303 """ 

304 Attributes defined for the Open edX Exam downstream effects. 

305 

306 Note that events that use this data type: 

307 A. Pretain to "Special Exams", e.g. Timed or Proctored exams, and not non-timed course 

308 subsections that are labelled as an exam. 

309 B. Are only ever emitted from the newer exams backend, edx-exams, and never from the 

310 legacy exams backend, edx-proctoring. 

311 

312 The event signals that use this data have the prefix `EXAM_`, which is equivalent to "special exam". 

313 We are using this as a shortened form of the prefix `SPECIAL_EXAM` to avoid confusion, as these are likely 

314 the only type of exams that will be of concern in the context of events/the event bus. 

315 

316 For more information, please see the ADR relating to this event data: 

317 https://github.com/openedx/openedx-events/blob/main/docs/decisions/0013-special-exam-submission-and-review-events.rst 

318 

319 Arguments: 

320 student_user (UserData): user object for the student to whom the exam attempt belongs 

321 course_key (CourseKey): identifier of the course to which the exam attempt belongs 

322 usage_key (UsageKey): identifier of the content that will get a exam attempt 

323 exam_type (str): type of exam that was taken (e.g. timed, proctored, etc.) 

324 requesting_user (UserData): user triggering the event (sometimes a non-learner, e.g. an instructor) 

325 """ 

326 

327 student_user = attr.ib(type=UserData) 

328 course_key = attr.ib(type=CourseKey) 

329 usage_key = attr.ib(type=UsageKey) 

330 exam_type = attr.ib(type=str) 

331 requesting_user = attr.ib(type=UserData, default=None)
